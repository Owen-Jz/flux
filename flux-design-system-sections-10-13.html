Section 10

Accessibility Standards

WCAG 2.2 AA compliance requirements and inclusive design guidelines

Color Contrast Requirements

All text and visual elements must meet minimum contrast ratios to ensure readability for users with low vision or color blindness.

4.5:1
Minimum for normal text

Body text under 18px regular weight

3:1
Minimum for large text

18px+ or 14px bold text

3:1
UI Components & Graphics

Interactive elements, focus indicators, chart elements

Compliant Example

Flux Purple (#7E3BE9) on White — Contrast ratio 5.2:1

Focus Indicators

All interactive elements must have clearly visible focus states that are never removed or hidden.

Implementation Standard

Use box-shadow: 0 0 0 3px var(--bg-primary), 0 0 0 6px var(--flux-purple) for a 3px purple focus ring. Never use outline: none without an alternative focus indicator.

Keyboard Navigation

All interactive elements must be fully accessible via keyboard with proper tab order and activation.

Tab Navigation

All interactive elements reachable via Tab key in logical reading order. Use tabindex="0" for custom controls.

Activation Keys

Enter and Space activate buttons and links. Escape closes modals. Arrow keys navigate menus and dropdowns.

Screen Reader Support

ARIA Labels

All icon-only buttons require aria-label or aria-labelledby. Example: aria-label="Close modal"

Heading Hierarchy

Logical order: h1 > h2 > h3. Never skip levels. Page title is h1, section titles h2, subsections h3.

Alt Text

All images require descriptive alt text. Use alt="" for decorative images only.

Reduced Motion

RESECTED

All animations must respect prefers-reduced-motion. Wrap animations in media queries to disable or reduce motion.

Color Blindness Safety

Never use color as the sole means of conveying information. Always pair with secondary indicators.

Good: Color + Icon + Text

Task status: ● Done with checkmark icon

Bad: Color Only

Task status: ● (no icon or label)

Touch Targets

44×44px
Minimum touch target size for mobile

All interactive elements on mobile must have at least 44x44px touch area. Add padding to small buttons to meet this requirement.

Cognitive Accessibility

Clear Labels

Use descriptive, actionable labels. "Delete task" instead of "Click here". Avoid jargon and abbreviations.

Error Messages

Include helpful suggestions: "Password must be 8+ characters with one number" instead of "Invalid password".

Consistent Navigation

Navigation placement and behavior stays consistent across all screens. Users learn patterns.

Clear Feedback

Show loading states, success confirmations, and error states clearly. Use inline validation.

Accessibility Checklist

  • ✓
    Color contrast ratios meet 4.5:1 (text) and 3:1 (large text/UI)
  • ✓
    All interactive elements have visible 3px purple focus indicators
  • ✓
    Full keyboard navigation with logical tab order
  • ✓
    ARIA labels on all icon buttons and interactive elements
  • ✓
    Proper heading hierarchy (h1 → h2 → h3)
  • ✓
    Alt text on all meaningful images
  • ✓
    Animations respect prefers-reduced-motion
  • ✓
    Color never used alone to convey information
  • ✓
    Touch targets minimum 44×44px on mobile
  • ✓
    Form errors include helpful suggestions

Social Color Do's and Don'ts

DO: Use purple as dominant brand color in at least 60% of posts. Ensure text has sufficient contrast. Maintain consistent color palette across all platforms.

DON'T: Use busy backgrounds that compete with brand. Overwhelm with multiple bright colors. Use color alone to convey information (accessibility).

Section 13

Presentation Templates

Slide deck guidelines and visual standards for presentations

Slide Deck Specifications

Aspect Ratio

16:9 Widescreen

1920 × 1080px (standard)

Margins

Maintain 10% margin from all slide edges. Keep content within "safe zone" to prevent cutoff on projection.

Typography Scale

48-60px — Slide Title

Used for main slide headlines. Font: Plus Jakarta Sans, Weight: 800

28-36px — Subtitle / Section Header

Used for section introductions or supporting headlines. Font: Plus Jakarta Sans, Weight: 600

18-24px — Body Text

Used for bullet points, descriptions, and body content. Font: Plus Jakarta Sans, Weight: 400/500

14-16px — Captions and Footnotes

Used for source citations, timestamps, and secondary information. Font: Plus Jakarta Sans, Weight: 400

Slide Layouts

Title Slide
Purple gradient background. Centered title (48-60px) + subtitle. Logo in top-left corner. No other elements.
Section Divider
Full purple background or white with large colored left bar. Section number + title only. Minimal, high impact.
Content with Bullets
White background. Title top-left (28-36px). 3-5 bullet points. Icon bullets optional. 40% width on left for illustration.
Two-Column Comparison
White background. Title across top. Two equal columns with headers "Option A" vs "Option B" or "Before" vs "After".
Image + Caption
Image fills 60% left side. Caption and credit on right. Title can overlay image or sit above. Ensure image is high resolution.
Data / Chart Slide
Title + subtitle top. Large chart or graph center. Key insight callout box bottom-right. Source citation footnote.
CTA Slide
Strong headline (action verb). Supporting line. Button or contact info. Minimal design, maximum focus on conversion.
Quote Slide
Large quotation marks in purple. Quote text centered, 28-36px italic. Attribution below. Photo optional (circular, left-aligned).

Pitch Deck Guidelines

Headers & Titles

  • Use Flux Purple for all headings
  • Keep titles to 5 words maximum
  • Use sentence case, not title case
  • Subtitles should complement, not repeat

White Space & Layout

  • Leave 40% of slide as white space
  • Align elements to invisible grid
  • Group related content together
  • Use consistent indentation for hierarchy

Color Application

  • 60% white/light gray backgrounds
  • 30% brand purple for emphasis
  • 10% accent colors (sparingly)
  • Never use more than 3 colors per slide

Consistency Rules

  • Same template for all slides in a section
  • Use the same position for titles
  • Bullet points aligned consistently
  • Logo in same position every slide

Icons for Presentations

Use the Lucide icon library for consistency. Icons must be at least 32px for readability in presentations.

Pro Tip: Icon Usage

When using icons in slides, apply the same weight and style throughout. Use outline style icons (2px stroke) for a clean, modern look. Always ensure icons have sufficient contrast against their background. For dark backgrounds, use white icons. For light backgrounds, use purple or dark gray icons.
