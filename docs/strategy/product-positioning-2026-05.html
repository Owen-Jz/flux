Positioning & Direction

Captured May 2026  ·  Branch: master  ·  Session: office-hours + multi-scale plan-with-ai implementation
The Core Shift
"Task management made simple."
"Describe your project. Flux plans it."
What This Actually Means
This is not a copy change. It's a category change. We moved from competing with Notion, Trello, and Linear (task managers) to competing with nothing — because no mainstream task tool generates your project structure from a description.

Why the Original Framing Wasn't Working

Task management for solo users is solved and commoditised. Notion free tier, Linear free tier, Todoist — all excellent and free. Four options were evaluated and rejected:

OptionWhy Rejected
Mobile-first experienceTable stakes. Every app is adding this. Not a moat.
Paystack + NGN pricingPricing alone isn't a product story. Doesn't survive competition.
Client-shareable boardsFigma, Notion, Trello all have this. Not differentiated.
AI decomposition (buried API)The hook existed — it just wasn't visible to anyone.
The Real Discovery
Flux already had a working AI decomposition engine (MINIMAX, POST /api/v1/tasks/decompose). The problem wasn't the product — it was that the capability was buried in an API call nobody knew existed.

Target User

Chosen Target
Nigerian freelance developer / designer managing client work.
Not "individuals." Not "solo users." A specific person with a specific pain.

Why This Person

  • You ARE this person. You know what frustrates them because you've lived it.
  • You can reach them directly — Nairaland, dev WhatsApp/Telegram groups, Twitter/X Nigerian tech.
  • They have a financially-consequential pain: disorganised client work leads to missed deliverables, embarrassment, and lost income.
  • Freelancers already budget for tools. They think of software as a business expense.

Their Status Quo

WhatsApp notes + scattered Google Docs + an abandoned Notion setup. The "plan" for each client project lives across 3 WhatsApp chats, a doc, and memory. When a client asks "where are we?" — that's the acute pain moment.

The Hook

The Pitch
"Tell Flux what you're building. It plans the project."

The AI is the mechanism, not the brand. The pitch is the outcome: describe your project in plain language, and a full board appears — tasks, priorities, time estimates.

Why This Is Actually Different

Every other task tool makes you build the structure yourself: create board → create columns → type each task → set each priority → estimate each task.

Flux collapses all of that into one step. This is not a 10% improvement. It's a different interaction model.

What We Built (May 2026)

Shipped · Phase 1

Surface the AI

Added "Plan with AI" CTA to empty board state. Exposed existing MINIMAX decomposition via UI (was API-only). Added deadline field. Updated modal header to "Plan with AI".

Shipped · Phase 2

Multi-Scale Planning Wizard

Full replacement of the old single-task decomposer. A 6-step wizard that generates entire project structures before creating anything.

Scope → Input → Planning → Review → Creating → Done

Scope Step

  • This board — generates tasks for the current board (existing behaviour, now accessible)
  • Full project — generates multiple boards with tasks (entirely new capability)

Review Step (key innovation)

Nothing is created until the user confirms. The review shows a full checklist — boards and tasks — that the user can edit before committing. Boards can be expanded/collapsed. Individual items can be deselected.

New Files Shipped

  • types/ai-plan.ts — shared interfaces with PlanScale alias and ConfirmedPlan discriminated union
  • lib/llm/project-planner.ts — board and project planning LLM prompts
  • app/api/ai/plan/route.ts — dry-run planning endpoint (no DB writes)
  • actions/ai-plan.ts — bulk creation with pre-flight plan-limit check
  • components/board/plan-with-ai-modal.tsx — the 6-step wizard component

The Agreed Premises

  1. The right individual target is the Nigerian freelance developer/designer managing client work — not a generic solo user.
  2. The status quo (WhatsApp + scattered docs + abandoned Notion) is the real competitor, not Linear or Trello.
  3. "Better task management" is not the pitch. "Describe your project, Flux plans it" is the pitch — it changes what category Flux competes in.
  4. The AI decomposition already existed and worked. The gap was making it the entry point, not a power-user API call.
  5. Individual willingness to pay is directly tied to economic value. Saving 45+ min of planning per project at 3–4 projects/month = clear ROI.
  6. Distribution before full repositioning — but the product must match the pitch before distribution runs. Otherwise, conversion from interest to signup is weak.

Distribution Strategy

Where to Reach the Target User

The Pitch Post

Headline
"I built the AI project planner I wish I had when I was taking on client work."
(Not: "I built a task manager.")

The Demo

Open a new board → describe a client project → AI creates the full board in seconds. This is showable. Show it.

Validation Targets Before Investing in Marketing

  • 10 freelancers agree to try Flux after seeing the demo
  • 3 of those use "Plan with AI" for a real client project
  • 1 says "this saved me time" in their own words, unprompted

Monetization Path

For Individuals

Free tier works as top-of-funnel. Individual willingness-to-pay is low unless the product has direct economic value. The AI planning feature creates that value specifically for freelancers.

The Real Money

Teams and workspaces. An individual freelancer signs up, uses it for client work, brings in a collaborator or the client. That's where conversion happens.

Suggested Gate (after 50 active users)
Free: 3 AI project plans/month
Paid: Unlimited AI planning + full project mode (multi-board generation)

Do not gate prematurely. Get the feature used first. You need evidence before you restrict access.

Open Questions

AI output quality with real users The MINIMAX integration has been tested via API but not observed with real freelancers describing projects in natural language. Quality needs to be validated with 3 real users before any marketing spend.
Homepage copy Still says generic task management. Should be updated to reflect the AI planner pitch — but only after validating it resonates in distribution week.
Adaptive workspace (Approach C) Revisit after 50+ signups with behavioral data showing meaningful differences by use case. Don't build from instinct.

What Was Noticed (Founder Signals)

⚡
You rejected the first four positioning options with specific reasoning, not vague dissatisfaction. "I feel they move offices" (mobile is table stakes). "Everybody can share both at this point." That's taste operating before the reasoning catches up.
🎯
You picked "Nigerian builder" without hesitation — didn't reach for the larger addressable market. You know who you're building for because you are that person. That specificity is the only version of this that works.
💡
You said the AI decomposition "should be one of the main selling factors" — you already had the right instinct. The issue wasn't identifying the hook, it was articulating it precisely.
🚀
